SlowQ

A macOS app that prevents accidental Cmd+Q — and a guided tour of how it works in Swift.

11 source files · ~950 lines of Swift · zero dependencies

The Big Picture

SlowQ is a menu-bar-only macOS app. It installs a system-level keyboard hook that intercepts Cmd+Q and requires you to hold the shortcut for a configurable delay before the frontmost app actually quits. No accidental tab closures, no lost work.

What is a Swift Package?

A Swift Package is the standard way to distribute and organize Swift code. The Package.swift manifest file (always at the project root) declares the package name, supported platforms, targets (libraries or executables), and dependencies.

SlowQ’s manifest declares a single executable target (SlowQ) requiring macOS 13+, plus a test target that pulls in the swift-testing library. There are zero runtime dependencies — everything uses Apple’s built-in frameworks.

App Entry Point — SlowQApp.swift

This is where everything begins. At just 29 lines, it’s the smallest file in the project, but it introduces several foundational Swift and SwiftUI concepts.

struct SlowQApp: App {
    @StateObject private var coordinator: AppCoordinator

    init() {
        _coordinator = StateObject(wrappedValue: AppCoordinator())

        _coordinator = StateObject(wrappedValue: coordinator)

@mainMarks the entry point of a Swift program. The annotated type must provide a static main() method or conform to a protocol that provides one (like App).

In Swift, @main tells the compiler “this is where the program starts.” The App protocol provides a static func main() behind the scenes, so you never write one yourself — SwiftUI handles the run loop, lifecycle, and event dispatch.

structA value type in Swift. Copied when assigned or passed. Preferred over class when you don't need reference semantics or inheritance. vs classA reference type in Swift. Passed by reference (shared). Supports inheritance. Used when identity matters or when conforming to certain protocols.

Swift has two main container types: structs (value types, copied on assignment) and classes (reference types, shared by pointer). SwiftUI views are always structs — they’re lightweight value descriptions that get recreated frequently. AppCoordinator is a class because it needs to be shared by reference across the whole app.

@StateObjectA property wrapper that creates and owns a reference to an ObservableObject. The object is created once and persists for the lifetime of the view.

@StateObject creates and owns a reference-type object. SwiftUI ensures it’s created exactly once and persists for the view’s lifetime. When the object’s @Published properties change, SwiftUI re-renders the views that depend on them.

MenuBarExtra & Scenes

A SwiftUI Scene is a top-level container that manages a window or UI region. MenuBarExtra creates a macOS menu bar icon with a dropdown panel. Settings { ... } creates the standard Preferences/Settings window that opens via Cmd+,. SlowQ has no dock icon or main window — it lives entirely in the menu bar.

import AppKit
import Combine
import SwiftUI

enum PermissionKind {
    case inputMonitoring
    case accessibility
}

@MainActor
final class AppCoordinator: ObservableObject {
    @Published var settings: SettingsStore
    @Published private(set) var inputMonitoringState: PermissionState
    @Published private(set) var accessibilityState: PermissionState
    @Published private(set) var isInterceptionRunning = false
    @Published var lastError: String?

    private let permissionService: any PermissionServicing
    private let launchAtLoginService: any LaunchAtLoginControlling
    private let interceptionService: QuitInterceptionControlling
    private var cancellables = Set<AnyCancellable>()

    // ... init with dependency injection (see full source)

    private func bindSettings() {
        settings.$isProtectionEnabled
            .dropFirst()
            .sink { [weak self] isEnabled in
                self?.reconcileInterception(isProtectionEnabled: isEnabled)
            }
            .store(in: &cancellables)
    }

    private func reconcileInterception(isProtectionEnabled: Bool) {
        guard isProtectionEnabled else {
            interceptionService.stop()
            isInterceptionRunning = false
            return
        }
        if interceptionService.start() {
            isInterceptionRunning = true
            clearInterceptionErrorIfNeeded()
        } else {
            interceptionService.stop()
            isInterceptionRunning = false
            lastError = Self.permissionErrorMessage
        }
    }
}

@MainActorEnsures all access to this type happens on the main thread. Required for UI work in Swift's structured concurrency model.

Swift’s concurrency model uses actors to prevent data races. @MainActor is a global actor that restricts all property access and method calls to the main thread — essential for anything touching UI. The compiler enforces this at build time.

ObservableObjectA protocol for classes whose changes should trigger SwiftUI view updates. Works with @Published properties and @ObservedObject/@StateObject. & @PublishedA property wrapper that automatically emits change notifications through the enclosing ObservableObject's objectWillChange publisher.

ObservableObject is a protocol for classes that broadcast changes. Any property marked @Published automatically fires a notification whenever its value changes, causing SwiftUI to re-render views that observe the object.

Combine: sinkA Combine subscriber that receives values from a publisher and executes a closure for each one. & AnyCancellableA type-erased cancellable object that cancels the subscription when deallocated. Store it to keep the subscription alive.

Combine is Apple’s reactive programming framework. The pattern here is: $isProtectionEnabled (the publisher for the property) → .dropFirst() (skip the initial value) → .sink { ... } (react to changes). The returned subscription is stored in a Set<AnyCancellable> so it stays alive for the lifetime of the coordinator.

Protocols & Dependency Injection

Notice any PermissionServicing and any LaunchAtLoginControlling — these are protocol types. The coordinator doesn’t know (or care) about the concrete implementations. This is dependency injection: the real services are passed in by default, but tests can substitute mocks. The any keyword is Swift’s existential type syntax — it means “any type that conforms to this protocol.”

Closures & [weak self]A capture list that holds a weak reference to self, preventing retain cycles when a closure is stored long-term.

Closures are anonymous functions. When a closure references self and is stored on that same object (as with Combine subscriptions), you get a retain cycle — neither can be freed. [weak self] breaks the cycle by holding a non-owning reference. Inside the closure, self? is optional (may be nil if the object was deallocated).

Settings — SettingsStore.swift

A compact persistence layer. Three settings, each backed by UserDefaults, each broadcasting changes via @Published.

SettingsStore.swift
@MainActor
final class SettingsStore: ObservableObject {
    static let defaultDelayMs = 1_000
    static let minDelayMs = 500
    static let maxDelayMs = 5_000

    private enum Keys {
        static let delayMs = "io.github.manas.SlowQ.delayMs"
        static let isProtectionEnabled = "io.github.manas.SlowQ.isProtectionEnabled"
        static let launchAtLogin = "io.github.manas.SlowQ.launchAtLogin"
    }

    private let defaults: UserDefaults

    @Published var delayMs: Int {
        didSet {
            let clamped = Self.clampDelay(delayMs)
            if clamped != delayMs {
                delayMs = clamped
                return
            }
            defaults.set(clamped, forKey: Keys.delayMs)
        }
    }

    @Published var isProtectionEnabled: Bool {
        didSet { defaults.set(isProtectionEnabled, forKey: Keys.isProtectionEnabled) }
    }

    @Published var launchAtLogin: Bool {
        didSet { defaults.set(launchAtLogin, forKey: Keys.launchAtLogin) }
    }

    init(defaults: UserDefaults = .standard) {
        self.defaults = defaults
        defaults.register(defaults: [
            Keys.delayMs: Self.defaultDelayMs,
            Keys.isProtectionEnabled: true,
            Keys.launchAtLogin: false,
        ])
        // ... read persisted values
    }
}

UserDefaultsA simple key-value persistence store provided by Apple. Ideal for small amounts of user preferences. Data persists across app launches.

UserDefaults is macOS/iOS’s built-in key-value store for lightweight preferences. It persists data to a plist file on disk automatically. .standard is the shared singleton, but the init accepts any UserDefaults instance — tests use an ephemeral one so they don’t pollute real settings.

Property Observers: didSetA property observer that runs immediately after the property's value changes. Commonly used for side effects like persistence or validation.

Swift lets you attach code to property changes. didSet runs after a value is assigned. Here, every time delayMs changes, the new value is clamped to [500, 5000] and persisted to UserDefaults. If the clamped value differs, the property is reassigned (which triggers didSet again with the corrected value).

staticA property or method that belongs to the type itself, not to instances. Accessed via TypeName.property rather than instance.property. properties

static let defaultDelayMs = 1_000 defines a constant that belongs to the type, not to any instance. Access it as SettingsStore.defaultDelayMs. Swift allows underscores in numeric literals for readability: 1_000 is just 1000.

Permissions — AccessibilityPermissionService.swift

macOS requires explicit user consent for apps that monitor keyboard events. SlowQ needs Input Monitoring (to see key presses) and Accessibility (to intercept and suppress them). The permission APIs differ between macOS versions.

protocol PermissionServicing {
    func preflightListenPermission() -> Bool
    func requestListenPermission() -> Bool
    func isAccessibilityTrusted() -> Bool
}

struct AccessibilityPermissionService {
    private let api: any AccessibilityPermissionAPI
    private let usesCGListenAccess: Bool

    private static var defaultUsesCGListenAccess: Bool {
        if #available(macOS 15.0, *) {
            return true
        }
        return false
    }

    func preflightListenPermission() -> Bool {
        if usesCGListenAccess {
            return api.preflightListenEventAccess()
        }
        return api.isProcessTrusted()
    }
}

extension AccessibilityPermissionService: PermissionServicing {}
AccessibilityPermissionAPI — the inner protocol & its concrete implementation

The permission service uses a two-layer abstraction: an inner protocol defines the raw macOS API surface, and a concrete struct wraps the real system calls. Tests mock the protocol to simulate permission states without touching real macOS APIs. 

protocol AccessibilityPermissionAPI {
    func preflightListenEventAccess() -> Bool
    func requestListenEventAccess() -> Bool
    func isProcessTrusted() -> Bool
    func isProcessTrustedWithPrompt() -> Bool
}

struct SystemAccessibilityPermissionAPI: AccessibilityPermissionAPI {
    func preflightListenEventAccess() -> Bool {
        CGPreflightListenEventAccess()
    }

    func requestListenEventAccess() -> Bool {
        CGRequestListenEventAccess()
    }

    func isProcessTrusted() -> Bool {
        AXIsProcessTrusted()
    }

    func isProcessTrustedWithPrompt() -> Bool {
        let promptKey = kAXTrustedCheckOptionPrompt.takeUnretainedValue() as String
        let options = [promptKey: true] as CFDictionary
        return AXIsProcessTrustedWithOptions(options)
    }
}

Extension Conformance

extension AccessibilityPermissionService: PermissionServicing {} adds protocol conformance in a separate block rather than on the original type declaration. This is a common Swift pattern: the struct defines its methods in the main body, then an extension declares “these methods satisfy the protocol.” Extensions can also add conformance to types you don’t own.

protocolDefines a blueprint of methods, properties, and requirements that conforming types must implement. Similar to interfaces in other languages.

A protocol is a contract — it declares what a type must do, not how. PermissionServicing requires three methods. Any type that implements them all can declare conformance. Here, AccessibilityPermissionService conforms via an extension (a way to add capabilities to an existing type in a separate block).

#availableA compile-time check that conditionally executes code based on the OS version. Prevents crashes on older systems that lack newer APIs.

macOS 15 introduced dedicated CGPreflightListenEventAccess / CGRequestListenEventAccess APIs. Older versions must use the accessibility API (AXIsProcessTrusted). #available(macOS 15.0, *) is a compile-time check that branches on the runtime OS version, ensuring the app works on macOS 13+ while using newer APIs when available.

@preconcurrencySuppresses strict Sendable checking for an import, used when a framework hasn't been updated for Swift 6 concurrency yet.

The file’s first line is @preconcurrency import ApplicationServices. Swift 6 enforces strict data-race safety (“Sendable” checking). Some Apple frameworks haven’t been fully annotated yet, so @preconcurrency tells the compiler: “trust this import for now, suppress concurrency warnings.”

Keyboard Interception — The Core Engine

The heart of SlowQ is split into three files, each with a single responsibility. This separation makes the complex event-handling logic testable in isolation.

Pure decision logic with no side effects. Given an input event and the current hold state, it returns one of five decisions. No timers, no system calls — just a function.

    [*] --> Evaluate
    Evaluate --> reenableTapAndPass: tap disabled
    Evaluate --> pass: non-keyboard event
    Evaluate --> TrackAppSwitcher: keyboard event

    TrackAppSwitcher --> pass: Cmd+Tab (switcher active)

enum InterceptionDecision: Equatable {
    case pass
    case swallow
    case reenableTapAndPass
    case beginHold
    case cancelHold
}

struct InterceptionPolicy {
    private(set) var appSwitcherActive = false

    mutating func decide(
        input: InterceptionInput,
        holdIsActive: Bool,
        shouldHandleCmdQ: () -> Bool
    ) -> InterceptionDecision {
        switch input.event {
        case .tapDisabledByTimeout, .tapDisabledByUserInput:
            return .reenableTapAndPass
        case .flagsChanged, .keyDown, .keyUp:
            break
        case .other:
            return .pass
        }

        // Track Cmd+Tab app switcher state
        if input.commandPressed && input.key == .tab {
            appSwitcherActive = true
        } else if !input.commandPressed {
            appSwitcherActive = false
        }

        // During hold: swallow Q presses, cancel on release
        if holdIsActive {
            if input.event == .keyUp && input.key == .q {
                return .cancelHold
            }
            if input.key == .q { return .swallow }
        }

        guard input.event == .keyDown else { return .pass }
        guard input.commandPressed, input.key == .q,
              !input.controlPressed else { return .pass }
        guard !appSwitcherActive else { return .pass }
        guard shouldHandleCmdQ() else { return .pass }

        return .beginHold
    }
}

mutatingRequired keyword on struct methods that modify the struct's properties. Since structs are value types, mutation requires explicit opt-in. methods on structs

Structs are value types — their methods can’t modify properties by default. The mutating keyword opts in to mutation. Here, decide() can set appSwitcherActive. This makes the mutation visible at the call site: the variable must be var, not let.

Enums & Pattern Matching

Swift’s enum is much more powerful than in most languages. Each case is a distinct value (not an integer). The switch statement must be exhaustive — the compiler ensures you handle every case. guard ... else { return } is Swift’s “early exit” pattern: if the condition fails, you must leave the scope.

A time-based state machine that tracks whether the user has held Cmd+Q long enough. It knows nothing about keyboards or system events — just timestamps and durations.

State Machine
stateDiagram-v2
    [*] --> idle
    idle --> holding: start()
    holding --> completed: tick() reaches 100%
    holding --> cancelled: cancel()
    completed --> idle: reset()
    cancelled --> idle: reset()
    cancelled --> holding: start()
    completed --> holding: start()
        
enum HoldState: Equatable {
    case idle
    case holding(startTimestamp: TimeInterval)
    case completed
    case cancelled
}

final class HoldToQuitEngine {
    typealias TimeProvider = () -> TimeInterval

    private let now: TimeProvider
    private(set) var state: HoldState = .idle

    init(now: @escaping TimeProvider = CACurrentMediaTime) {
        self.now = now
    }

    var isHolding: Bool {
        if case .holding = state { return true }
        return false
    }

    func tick(delayMs: Int) -> HoldTickResult {
        guard case let .holding(startTimestamp) = state else {
            return .notHolding
        }
        let elapsedMs = max((now() - startTimestamp) * 1_000.0, 0)
        let progress = min(elapsedMs / Double(delayMs), 1.0)
        if progress >= 1.0 {
            state = .completed
            return .completed
        }
        return .holding(progress: progress)
    }
}

Enums with Associated Values

.holding(startTimestamp: TimeInterval) is an enum case that carries data. Unlike traditional enums that are just labels, Swift enums can store different data per case. guard case let .holding(startTimestamp) = state both checks the case and extracts the associated value in one step — this is pattern matching.

typealiasCreates a named alias for an existing type. Makes complex type signatures (especially closures) more readable.

typealias TimeProvider = () -> TimeInterval gives a readable name to the closure type “function that takes nothing and returns a TimeInterval.” The engine’s default time provider is CACurrentMediaTime (a monotonic clock), but tests inject a fake one to control time precisely.

CGEvent tap setup (the most complex code in the project) 
func start() -> Bool {
    guard eventTap == nil else { return true }

    let eventMask =
        mask(for: .flagsChanged)
        | mask(for: .keyDown)
        | mask(for: .keyUp)

    let callback: CGEventTapCallBack = { _, type, event, userInfo in
        guard let userInfo else { return nil }
        let service = Unmanaged<QuitInterceptionService>
            .fromOpaque(userInfo).takeUnretainedValue()
        return service.handleEvent(type: type, event: event)
    }

    guard let tap = CGEvent.tapCreate(
        tap: .cgSessionEventTap,
        place: .headInsertEventTap,
        options: .defaultTap,
        eventsOfInterest: eventMask,
        callback: callback,
        userInfo: Unmanaged.passUnretained(self).toOpaque()
    ) else { return false }

    let source = CFMachPortCreateRunLoopSource(
        kCFAllocatorDefault, tap, 0)
    CFRunLoopAddSource(CFRunLoopGetMain(), source, .commonModes)
    CGEvent.tapEnable(tap: tap, enable: true)

    eventTap = tap
    eventTapSource = source
    return true
}

CGEvent Taps

CGEvent.tapCreate is a Core Graphics API that installs a callback at the system level, receiving raw keyboard and mouse events before they reach any application. Returning nil from the callback swallows the event (the app never sees it). Returning the event passes it through. This is how SlowQ blocks Cmd+Q.

UnmanagedA wrapper for working with Core Foundation APIs that don't participate in Swift's automatic reference counting. Used to pass Swift objects through C callbacks. Pointers

The CGEvent tap callback is a C-style function pointer — it can’t capture Swift context. To pass self through the userInfo void pointer, the code uses Unmanaged.passUnretained(self).toOpaque() (convert to raw pointer) and Unmanaged.fromOpaque(userInfo).takeUnretainedValue() (convert back). “Unretained” means no extra reference counting — the service object must outlive the tap.

DispatchSourceTimer

While Cmd+Q is held, a DispatchSourceTimer fires every 16ms (~60fps) to tick the hold engine and update the overlay progress bar. The timer runs on the main queue since it drives UI updates. It’s created via a factory closure so tests can substitute a manual timer.

FrostRailProgressView is a private final class NSView subclass that draws the progress bar by overriding draw(_ dirtyRect:) — rendering a translucent rail and a filled portion proportional to progress. The state struct exposes mutating func updateProgress(_:) and mutating func hideAndReset() to drive transitions.

NSPanel & NSVisualEffectView

NSPanel is a special type of window designed for floating palettes and HUDs. The .nonactivatingPanel style mask means clicking it won’t steal focus from the frontmost app. NSVisualEffectView with .hudWindow material provides the system’s native frosted-glass blur effect — no custom shaders needed.

Custom NSView Drawing

FrostRailProgressView is a custom NSView subclass that draws the progress bar by overriding draw(_ dirtyRect:). It draws a translucent rounded-rect “rail” background, then a filled portion proportional to progress (0.0 to 1.0). Setting needsDisplay = true in the property observer tells AppKit to schedule a redraw.

NSLayoutConstraint (Auto Layout)

The overlay uses Auto Layout for positioning. Views are added to an NSStackView (vertical stack), then pinned to the parent using constraints: stack.leadingAnchor.constraint(equalTo: material.leadingAnchor, constant: 20). translatesAutoresizingMaskIntoConstraints = false is required to opt into constraint-based layout.

                    Label("Grant Access", ...)
                }
            }
        }
    }
}

@ObservedObjectA property wrapper for observing an ObservableObject that is owned elsewhere. The view re-renders when @Published properties change.

Unlike @StateObject (which creates and owns the object), @ObservedObject observes but doesn’t own an object created elsewhere. Both views receive the coordinator from their parent and watch it for changes.

BindingA two-way connection to a value owned elsewhere. Changes flow both ways — the UI reads and writes through the binding.

The slider needs a Binding<Double> but delayMs is an Int. A custom Binding bridges the types: get converts Int→Double, set rounds and converts back. This is a common pattern for adapting SwiftUI controls to your data model.

@ViewBuilderA result builder that lets you write multiple view expressions in a closure and combines them into a single composite view.

@ViewBuilder is a result builder — it lets you write multiple view expressions inside a function body and automatically combines them. Without it, the permissionRow method couldn’t use if/else to conditionally produce different views.

Launch at Login — LaunchAtLoginManager.swift

A thin wrapper around Apple’s SMAppService API, split into two protocol layers for testability.

LaunchAtLoginManager.swift
protocol LaunchAtLoginControlling {
    var isEnabled: Bool { get }
    func setEnabled(_ enabled: Bool) throws
}

struct LaunchAtLoginService: LaunchAtLoginControlling {
    private let service: any MainAppServiceControlling

    var isEnabled: Bool {
        service.status == .enabled
    }

    func setEnabled(_ enabled: Bool) throws {
        if enabled {
            try service.register()
        } else {
            try service.unregister()
        }
    }
}

struct ServiceManagementMainAppService: MainAppServiceControlling {
    func register() throws {
        guard #available(macOS 13.0, *) else { return }
        try SMAppService.mainApp.register()
    }
}

SMAppService

SMAppService.mainApp (macOS 13+) is the modern API for registering an app to launch at login. It replaces the older SMLoginItemSetEnabled approach. Calling .register() adds the app to Login Items; .unregister() removes it.

throwsMarks a function that can throw an error. Callers must use try/try?/try! and optionally catch errors. & Error Handling

Functions marked throws can fail with an error. Callers use try and can wrap calls in do { ... } catch { ... } blocks. In the coordinator, errors from setEnabled are caught and displayed as lastError. Swift doesn’t have checked exceptions — any function can throw any Error type.

Testing Patterns

SlowQ uses Swift Testing (the modern testing framework) rather than the older XCTest. Every service has a protocol layer so tests can inject mocks. The project has 9 test suites:

  • AccessibilityPermissionServiceTests — permission check branching (macOS 15 vs legacy)
  • AppCoordinatorTests — coordinator wiring, permission states, interception lifecycle
  • HoldToQuitEngineTests — time-based state machine transitions
  • InterceptionPolicyTests — pure-logic decision table
  • LaunchAtLoginServiceTests — register/unregister via mock SMAppService
  • OverlayStateTests — state mutations, clamping, visibility
  • QuitInterceptionServiceTests — full integration with mock overlay, engine, and timer
  • SettingsStoreTests — persistence round-trips, clamping, defaults
  • UISmokeTests — SwiftUI view instantiation without crashes

private final class MockOverlayController: OverlayPresenting {

    var updateCalls: [Double] = []

    }

}

@testable importImports a module with elevated access — internal declarations become visible to the test target. Only works within test targets.

Swift defaults to internal access (visible within the module). Test targets are separate modules, so they normally can’t see internal types. @testable import SlowQ opens up internal access for testing without making anything public.

Swift Testing Framework

The modern Swift Testing framework uses @Suite to group tests and @Test to mark individual test functions. Assertions use the #expect(...) macro, which provides rich failure messages showing both sides of the comparison. Unlike XCTest, test structs don’t need to subclass anything.

Mock Objects

Because every service dependency is behind a protocol, tests create lightweight mock classes that record method calls. MockOverlayController conforms to OverlayPresenting and stores every call in arrays. Tests can then assert: “was show() called with the right app name?”, “was hideAndReset() called after cancel?”.

Isolated UserDefaults

Both AppCoordinatorTests and UISmokeTests create an ephemeral UserDefaults suite per test to avoid cross-test pollution: 

let suiteName = "io.github.manas.SlowQ.tests.\(UUID().uuidString)"
guard let defaults = UserDefaults(suiteName: suiteName) else {
    fatalError("Failed to create isolated defaults suite")
}
return SettingsStore(defaults: defaults)

Each test run gets a unique suite name (via UUID), so persisted values from one test never leak into another. This works because SettingsStore.init accepts any UserDefaults instance — the dependency injection pattern pays off.
